# message-library.md

## Purpose

Full library of messages available in the service.  
Message content is separated from system specification so this list can expand without changing core mechanics.

## 目的

サービスで使用されるメッセージの完全ライブラリ。  
メッセージ内容をシステム仕様から分離し、将来メッセージが増えてもコア仕様を変更せずに拡張できるようにする。

---

## Design Principles

Messages should be short, calm, and emotionally neutral or gentle.  
They represent quiet presence rather than conversation.  
Avoid strong opinions, arguments, or anything that breaks the atmosphere of a quiet night.

## デザイン原則

メッセージは短く、穏やかで、感情的に強すぎないものにする。  
目的は会話ではなく「静かな存在の共有」である。  
強い主張や議論を生む内容、夜の雰囲気を壊すものは避ける。

## Expansion Rule

New messages may be added in future updates.  
All messages defined in previous project documents must be preserved when expanding this list.

## 拡張ルール

将来のアップデートでメッセージは追加される可能性がある。  
過去のプロジェクトドキュメントに存在するメッセージは、このライブラリ拡張時にも必ず保持する。
